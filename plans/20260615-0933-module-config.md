> **Child of** `plans/20260615-0548-gh-optivem-modular-monolith-parent.md`. This is child plan **#4 (Config)** in the modular-monolith decomposition. Pure moves only — no behavior change to the CLI.

## TL;DR

**Why:** The Config concern (`config`, `configinit`, `projectconfig`) is still flat at `internal/`, and `projectconfig` — imported by 40 files across *every* module — has no clean home. With seam #1 resolved it no longer imports the engine, so it is finally kernel-eligible. Until this lands, the Process carve-out (#7, the last and hardest child) can't proceed against a clean Config module.

**End result:** `projectconfig` lives in the shared kernel (`internal/kernel/projectconfig`), so the 40 cross-module imports are legal "depend-down" edges and no module reaches into Config's internals. `configinit` is nested under `internal/config/configinit` alongside the existing `internal/config/optivemyaml` leaf; `config` stays at `internal/config`. `go build ./...` and `go test ./...` stay green; the CLI behaves identically.

## Outcomes

What we get out of this — the goals and deliverables:

- **`projectconfig` promoted to the shared kernel** at `internal/kernel/projectconfig`. Every module legally depends on the kernel, so its 40 importers (in `atdd`, `build`, `devworkflow`, `scaffolding`, `config`, `configinit`, `main.go`) become clean downward edges instead of cross-module reach-ins.
- **`configinit` nested** at `internal/config/configinit`, alongside `internal/config/optivemyaml`. The Config module is now `internal/config/**`.
- **`config` unchanged** at `internal/config` — its ~25 importers need no edits.
- **Regression-safe**: no behavior change. `go build ./...` && `go test ./...` green after every move.
- **Import guard preserved/strengthened**: `projectconfig`'s `import_guard_test.go` moves with it and continues to assert it imports nothing from `internal/atdd/**` (and ideally nothing outside the kernel — it's a leaf).
- **Process carve-out (#7) unblocked** — the last child can plan against a finished Config module.
