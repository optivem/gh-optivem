### Decisions taken during execution (differ from the original draft)

- **Env var name:** `GH_OPTIVEM_RUN_WIP_TESTS` (not `ATDD_RUN_WIP`) — matches the `GH_OPTIVEM_` convention and drops user-visible "ATDD" jargon. Value is the exact string `1`.
- **Injection (Item 3):** fully internal — `actions.runCommand` sets the var (with `defer` restore) on its `isTestRun` branch only, so the child `gh optivem test run` → `mvn`/`dotnet`/`playwright` inherits it. **No `--wip` CLI flag**, nothing in `--help`.
- **C# gate (Item 1):** `[SkippableFact]` + `Skip.IfNot(...)` via the `Xunit.SkippableFact` package; the package was added to `shop/system-test/dotnet/SystemTests/SystemTests.csproj` (xUnit v2 stack — no native runtime skip).
- **`renderGateMarkerExample`** takes only `lang` (the permanent gate is ticket-independent — no `ticketID`/`issueTitle`).
- **Item 9 corrections:** docs live at `internal/assets/runtime/references/code/language-equivalents/{java,csharp,typescript}.md` (+ `README.md`), not `docs/atdd/code/...`; `architecture.yaml` had no `test-disabler`/`test-enabler` nodes (no-op there).
- Skip-message wording drops "ATDD": `Work-in-progress test; set GH_OPTIVEM_RUN_WIP_TESTS=1 to run`.

## Out of scope

- **Stripping the gate at ticket close.** Recommended permanent gate. If you want the AT to join regular post-merge CI without an opt-in env var, add a step that introduces a `strip-gate` step at the end of `change-system-behavior` (after REFACTOR_OPPORTUNISTICALLY) — one file edit per ticket, still net win vs 6 today.
- **Tag-based filtering instead of env-var gating.** Env var is uniform across the three languages; tag filtering would need per-project config propagated to the `shop` template and every scaffolded repo.
- **CT-side gating.** The gate applies only to AT methods the writer authors; if CT-side gating is needed later, the CT writer can reuse `renderGateMarkerExample`.
- **xUnit v3 migration.** Deliberately deferred — `Xunit.SkippableFact` keeps the shop on its existing xUnit v2 stack. Revisit as a separate modernization if a native `Assert.Skip*` is wanted.
- **Diagram regeneration.** Auto-regenerated by CI (`regenerate-diagram` workflow); `docs/process-diagram.md` + `docs/images/*.svg` still reference the removed nodes until that runs on push to main.
