# CONTRACT TEST CYCLE

> 🚧 **Work in progress.** This doc (and its sibling [`atdd-at-cycle.md`](atdd-at-cycle.md)) is being built up to become the canonical CT-cycle process spec, to be referenced by the per-phase agent prompts under `internal/assets/runtime/prompts/atdd/` (the `ct-*.md` prompts). The goal is to make those prompts significantly leaner — each prompt should reference the relevant section of this doc rather than restating the cycle rules inline. Once this doc reaches parity with the global process pages (`internal/assets/global/docs/atdd/process/ct-*.md`), those four global pages will be deleted and the runtime prompts will be slimmed to a thin pointer + per-phase operational notes.

The CT (Contract Test) cycle is a sub-cycle of the AT cycle, invoked from `AT-RED-EXTERNAL-SYSTEM-DRIVER` when new external system driver interface methods are needed. It uses the same `@Disabled` reason convention, phase-output flag schema, and phase scope policy as AT — see [`atdd-at-cycle.md` → Conventions](atdd-at-cycle.md#conventions).
