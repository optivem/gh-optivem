# CONTRACT TEST CYCLE

The CT (Contract Test) cycle is a sub-cycle of the AT cycle, invoked from `AT-RED-EXTERNAL-SYSTEM-DRIVER` when new external system driver interface methods are needed. It uses the same `@Disabled` reason convention, phase-output flag schema, and phase scope policy as AT — see [`at-cycle.md` → Conventions](at-cycle.md#conventions).
