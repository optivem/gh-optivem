# CONTRACT TEST CYCLE

The CT (Contract Test) cycle is a sub-cycle of the AT cycle, invoked from `AT-RED-EXTERNAL-SYSTEM-DRIVER` when new external system driver interface methods are needed. It uses the same disable/enable test conventions ([disable-tests](../shared/disable-tests.md), [enable-tests](../shared/enable-tests.md)) and the same [phase scope policy](../../../shared/conventions.md#phase-scope-policy) as AT.
